Do note that the %types.r table (WIP version) encodes the type/antiform information as well: Such as for splice:

group       "list that evaluates expressions as an isolated group"
~splice~    "fragment of multiple values without a surrounding block"    
            (node1)
            [any-series? any-sequencable?]

And for error:

warning     "context with id, arguments, and stack origin"
~error~:U   "error state that is escalated to a panic if not triaged"
            (node1 node2)
            [any-inert?]

(The :U means the antiform is "unstable".)

In any case, %types.r is a pretty good example of dialecting! As much as possible, it's good to use Rebol to do code generation for properties in the C (and also to put in the binary for HELP)!

%types.r has come a long way since R3-Alpha!
