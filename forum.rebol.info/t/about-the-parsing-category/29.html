One of the most popular and developed dialects in Rebol is the PARSE dialect.

  • Original Introduction in Rebol2 Documentation

  • Red's writeup when adding the feature

  • Old PARSE Tutorial from what was codeconscious.com

Some evolutions of the PARSE dialect were introduced by a spin-off called "Topaz", which added the idea of parse expressions synthesizing a value--in addition to advancing the input. These values could then be captured by SET-WORD! (instead of capturing the parse position in the set word variable, as the original PARSE did).

Ren-C more aggressively evolved the dialect to be based on functions that could be written in userspace--a parallel to "parser combinators". This implementation is called UPARSE:

Introducing The Hackable Usermode PARSE ("UPARSE") Parsing
!!! BIG NEWS !!! I've griped about historical PARSE's weaknesses for some time. To name a few: Users can't add new keywords, or tweak existing ones. It's a monolithic chunk of C code that you have to recompile to get any changes to. And since Rebol2's heyday, there have arisen a competitive landscape of "parser combinators" which allow users to build new parse "keywords" and compose parsers out of each other.
